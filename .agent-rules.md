# Agent Rules — ao-dashboard

## Core constraints
- Never rename the spawn branch after `ao spawn`.
- One issue = one branch = one PR.
- Before every push: run lint, typecheck, targeted tests.
- Do not touch CI, infra, or systemd service files unless the issue explicitly allows it.
- If CI fails twice, stop autopatching and post a root-cause summary.
- Use conventional commits (`feat:`, `fix:`, `chore:`) and reference the issue number.
- Do not force-push after reviewer comments unless rebasing is required.

## PR review + merge policy (MANDATORY)

AO/Codex writes code. Архимед reviews. КАЖДЫЙ PR проходит ВСЕ шаги ниже.

### Review checklist (выполнять по порядку, не пропускать)

1. **Read contract** — `cat ~/clawd/tasks/<task_id>/contract.json` + issue body. Понять: что должно быть сделано, acceptance criteria, success_definition.

2. **Read design** — проверить соответствие дизайну Лео:
   - Все цвета из `design-tokens.json` (`bg-void`, `bg-surface`, `accent-amber`, etc.)
   - НЕТ захардкоженных цветов (no `#1a1a2e`, no `bg-red-500`)
   - Шрифты: Inter (UI) + Geist Mono (code/logs)
   - Spacing, radius, animations по токенам

3. **Check diff for spec conformance** — по acceptance criteria из issue:
   - Каждый checkbox AC → есть ли реализация в diff?
   - Missing tests для новых code paths?
   - Unsafe shortcuts или TODOs?
   - Contract drift (scope beyond issue)?

4. **Write review-findings.json** — в `~/clawd/tasks/<task_id>/review-findings/pr-<N>-issue-<N>.json`:
   ```json
   {
     "schema_version": "1.0",
     "task_id": "tsk_...",
     "issue_id": "gh_N",
     "pr": N,
     "reviewer": "archimedes",
     "verdict": "approve|changes_requested|blocked",
     "findings": [{"id":"CR-001","severity":"critical|major|minor","file":"...","summary":"...","evidence":"...","required_fix":"..."}],
     "reviewed_at": "ISO-8601"
   }
   ```

5. **Post GitHub review comment** — ОБЯЗАТЕЛЬНО:
   ```bash
   gh pr review <PR> -R aerbaser/ao-dashboard --comment --body "## Code Review — <task_id>
   
   **Verdict:** approve|changes_requested
   **Findings:** N (critical: X, major: Y, minor: Z)
   
   [findings table if any]
   
   _Reviewed by archimedes via code-review-gate_"
   ```

6. **Decision:**
   - 0 critical + 0 major → `gh pr merge <PR> -R aerbaser/ao-dashboard --merge`
   - Any critical/major → changes_requested → fix via AO → re-review
   - НИКОГДА не мерджить без Steps 1-5

## Forbidden services
- Check `~/clawd/memory/forbidden-services.json` before any systemctl start.

## Design
- All colors and spacing MUST come from design tokens in `design-tokens.json`.
- Do not hardcode palette values.
- Import tokens through `src/index.css` or `tailwind.config.js`.

## API
- Backend is DRY: wrap existing CLI tools (ao-dashboard.js, task-store.js, status-synthesizer.js).
- Do not re-implement business logic — call the CLIs.
