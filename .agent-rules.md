# Agent Rules — ao-dashboard

## Core constraints
- Never rename the spawn branch after `ao spawn`.
- One issue = one branch = one PR.
- Before every push: run lint, typecheck, targeted tests.
- Do not touch CI, infra, or systemd service files unless the issue explicitly allows it.
- If CI fails twice, stop autopatching and post a root-cause summary.
- Use conventional commits (`feat:`, `fix:`, `chore:`) and reference the issue number.
- Do not force-push after reviewer comments unless rebasing is required.

## Forbidden services
- Check `~/clawd/memory/forbidden-services.json` before any systemctl start.

## Design
- All colors and spacing MUST come from design tokens in `design-tokens.json`.
- Do not hardcode palette values.
- Import tokens through `src/index.css` or `tailwind.config.js`.

## API
- Backend is DRY: wrap existing CLI tools (ao-dashboard.js, task-store.js, status-synthesizer.js).
- Do not re-implement business logic — call the CLIs.
